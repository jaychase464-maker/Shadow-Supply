# v0.9.2 — Counterfeit Press Drop-Zone Hotfix

## Problem

Correctly dragging Blank Note Stock onto the Paper Tray caused the material
to snap back and register a mistake.

The draggable material floats above the table for visibility. The old code
used `Bounds.Contains` against the thin three-dimensional tray collider.
This required the material's center to be inside the tray vertically, which
was impossible while dragging at the configured hover height.

## Fix

Drop validation now:

- Converts the dragged material position into the target zone's local space
- Checks only the horizontal X/Z footprint
- Ignores vertical Y height
- Respects rotated workbenches and process surfaces
- Adds a small forgiveness margin around each drop zone

This applies to:

- Paper Tray
- Pigment Bay
- Security Film Bed
- Cutting Mat
- Packaging Zone

## Installation

1. Exit Play Mode without saving the broken test run.
2. Close Unity.
3. Extract the ZIP into the Unity project root.
4. Replace all prompted files.
5. Reopen Unity and wait for compilation.

Do not rerun the Milestone 9A setup menu.

## First test

1. Start a fresh counterfeit run.
2. Drag the small yellow Blank Note Stock block.
3. Release it over the large orange Paper Tray.
4. It should snap into the tray and advance to step 2/9.
